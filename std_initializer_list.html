Introduction to std::initializer_list

std::initializer_list?

  • With a template parameter std::initializer_list<T>
  • a lightweight proxy object that provides access to an array of objects of type const T.
  • Available since C++11

No way to set elements?

  • The compiler will automatically construct a non-empty object of this class template type whenever an initializer list expression needs to be passed or copied
  • This is the only way to set the values of an object of type initializer_list

So, what on earth is that?

It is a list

As its name implies, it is a finite list of homogeneous type with sequence.

Its size is fixed

As initializer list expression is the only way to set the elements, its size is always known at compile time.

It is not an array

It does not own or manage its elements.

Cannot append/delete/insert/alter those.

Wait a minute...

Why is it passed by value? Isn't it expensive?

It's just OK

  • Copying an initializer list does not copy the underlying elements (standard, section 18.9)
  • The underlying array is not guaranteed to exist after the lifetime of the original initializer list object has ended (cppreference)

Explain like I'm five

  • Simple implementation of initializer_list would be
    • A pair of pointers that point to first/last
    • A pointer and a length
  • That is why the underlying elements' lifetime matters
